## Codebase Patterns
- TanStack Router root route uses `component` (for layout with `<Outlet />`) and `shellComponent` (for HTML document wrapper)
- Neo Brutalism style: bold borders (2-4px black), flat colors (yellow-300 header, amber-50 bg), no rounded corners, extrabold fonts
- Convex schema uses `defineSchema`/`defineTable` from `convex/server`, validators from `convex/values` (`v.string()`, `v.optional()`, etc.)
- Convex functions use `query`/`mutation`/`action` from `convex/_generated/server` with explicit `args` validators
- Convex actions support `fetch`, `AbortController`, `setTimeout` ŌĆö use for HTTP calls and side effects
- Tabs for indentation, double quotes (Biome formatter)
- Pre-existing lint errors in `vite.config.ts`, `src/components/Header.tsx`, `src/integrations/convex/provider.tsx` ŌĆö not introduced by this work
- Convex React hooks: `useAction` for actions, `useMutation` for mutations, `useQuery` for queries ŌĆö all from `convex/react`
- API references: import `api` from `../../convex/_generated/api` (or use path alias) ŌĆö auto-generated types reference functions by module.name (e.g., `api.links.fetchMetadata`)
- Demo files were in `src/routes/demo/` and `convex/todos.ts` ŌĆö both removed as part of schema cleanup
- Removing a table from schema requires also removing/updating all files that reference it for typecheck to pass
- Use `Doc<"tableName">` from `convex/_generated/dataModel` to explicitly type Convex document parameters when TS inference fails
